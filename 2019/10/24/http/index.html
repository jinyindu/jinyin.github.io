当我我们输入网址后发生了什么

  1. 输入网址并回车
  2. 解析域名
  3. 浏览器发生http请求
  4. 服务器处理请求
  5. 服务器返回HTML响应
  6. 浏览器处理HTML页面
  7. 继续请求其他资源

HTTP协议详解

  • HTTP是超文本传输协议，从www浏览器传输到本地浏览器的一种协议，网站是基于HTTP协议的
  • HTTP协议是由客户机到服务器请求和从服务器到客户机的响应进行约束和规范。

HTTP的工作过程

  • 一次HTTP操作称之为一个事物，工作过程分为四步
  1. 首先客户机与服务器需要建立连接，只要单击某个超链接，HTTP的工作开始。
  2. 建立连接后，客户机发送一个请求给服务器，请求的格式为：统一资源标识符（URL），协议版本号，后边是MIMEMIME信息包括请求修饰符，客户机信息的内容
  3. 服务器接收到请求后，给予相应的响应信息，其格式为一个状态行，包括信息的协议版本号，一个成功或错误的代码，后边是MIME信息包括服务器信息，实体信息。
  4. 客户端收到服务器返回的信息通过浏览器显示在用户的界面上，然后客户机与服务器断开连接。

请求与响应

  1. HTTP请求组成： 请求行，消息报头，请求正文
  2. HTTP响应组成： 状态行，消息报头，响应正文
  3. 请求行组成： 以一个方法符号开头，后面跟着请求的URL和协议的版本
  4. 状态行组成： 服务器HTTP协议的版本，服务器发回的响应状态代码和状态代码的文本描述

HTTP缓存机制

  1. 缓存会根据请求保存输出内容的副本，例如html页面，图片，文件，当下个请求来到的时候；如果是相同的URL，缓存直接使用副本响应访问请求，而不是向源服务器再次发生请求。
  2. 缓存的优点
  3. 减少相应的延迟
  4. 减少网络宽带消耗

HTTPS协议分析

  1. HTTPS协议的安全性有SSL协议实现，当前使用的TLS协议包含了4个核心子协议：握手协议，密钥配置切换协议，应用数据协议及报警协议。
  2. 数字证书：数字证书是互联网通信中标识双方身份信息的数字文件，🈶️CA签发
  3. CA：是数字证书的签发机构
  4. HTTPS协议，SSL协议，TLS协议，握手协议的关系
    1） HTTPS 可理解为基于SSL的HTTP协议，HTTPS协议安全是由SSL协议实现的。
    2） SSL协议是一种记录协议，扩展性良好，可以很方便的添加子协议，而握手协议便是SSL协议的一个子协议
